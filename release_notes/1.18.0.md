# Vibepollo 1.18.0 - 2026-07-11

Notice: Vibepollo may trigger false-positive antivirus alerts.

## Changes
- Virtual-display streams now run at 4x the requested refresh rate and receive an automatic low-latency frame cap for smoother pacing, including frame-generation streams.
- Frame-generation apps now prefer virtual-display capture by default, which captures generated frames correctly while retaining physical-display support for compatible setups.
- Fresh installs now recommend the Vibepollo Display Driver with per-client virtual displays. Windows 11 and Windows 10 receive OS-appropriate defaults, and SudoVDA remains available as a legacy option.
- Reworked the virtual-display driver, frame-limiter, and frame-generation guidance throughout the installer and Web UI, including clearer Windows 10 compatibility warnings.
- Added runtime bitrate adjustment and ABR capability negotiation for supported clients. NVENC changes bitrate without interrupting the stream; other encoders restart their encode session as needed.
- Application artwork now uses versioned IDs so updated cover art propagates reliably to clients.
- Playnite synchronization now waits for a complete library snapshot, avoiding partial sync results, and detects changed cover art for reliable client updates.
- Improved the Web UI on narrow screens, including the Audio/Video, Playnite, network, API-token, and override-setting flows.

## Fixes
- Made installer upgrades and downgrades transactional, so failed replacements roll back to the previously installed version instead of leaving Vibepollo uninstalled.
- Hardened virtual-display driver installation and recovery: failed revival no longer rolls back a completed install, stale runtime processes are recycled, and package registration is verified before setup reports success.
- Stabilized HDR streaming: virtual-display reinitialization no longer downgrades an HDR session, duplicate HDR mode messages no longer cause unnecessary TV resynchronization, and preferred Main10 SDR streams no longer trigger HDR/PQ display mode.
- Preserved numeric-backed values in app and client override dropdowns.
- Prevented manual Playnite sync from removing apps when its library snapshot is empty or incomplete.
- Restored the requested display configuration after an application exits and the last stream ends, even when revert-on-disconnect is disabled.
- Stopped Windows Graphics Capture from repeating stale frames after a dropped pacing interval.
- Fixed stale tray notifications replaying after taskbar re-registration and duplicate update notifications accumulating in Action Center.
- Made runtime language changes apply correctly and recovered from corrupt or permission-blocked configuration state files.
