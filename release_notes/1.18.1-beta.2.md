# Vibepollo 1.18.1-beta.2 - 2026-07-16

Notice: Vibepollo may trigger false-positive antivirus alerts. Details and workarounds are documented here: https://github.com/Nonary/Vibepollo/issues/29

## Changes
- Reduced Windows Graphics Capture overhead by eliminating redundant VRAM copies, lowering helper polling and scheduling pressure, and using dirty-region, pool-backed texture handoffs for efficient multi-client capture.

## Fixes
- Recovered owned temporary virtual displays securely after host restarts without deleting displays belonging to another process, preserved display identity through recovery and sleep, and stopped an absent Playnite service from delaying startup.
- Honored explicit client HDR requests when 10-bit SDR was preferred while keeping forced-SDR policy authoritative.
- Localized virtual capture, settings, application editing, Lossless Scaling, WebRTC, dashboard, and save-state workflows across supported languages, including live checkbox updates after locale changes.
- Reliably removed stale virtual display driver packages during upgrades and retried cleanup while the Windows DriverStore settled.
- Prevented RTSS reconnect recovery from replaying stale frame-limiter values.
- Forwarded Vibepollo session environment variables through Playnite fullscreen launches and restored Playnite's original environment after disconnecting.
- Restored the pre-launch screen saver setting after streaming pauses or terminates.
